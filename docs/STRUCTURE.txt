Codex Cortex Structure
======================

Naming Goals
------------

The project uses short names for installed files because future prompts,
agents, scripts, and humans need to reference them repeatedly.

The installed folder is `.cortex` because it is the project-local memory,
coordination, and recovery layer.

Installed Files
---------------

AGENTS.md
  Agent, prompt, and thread instructions.

START_HERE.md
  First-read file for future runs.

.cortex/README.md
  Index for the Cortex state folder.

.cortex/STATE.md
  Current state and next action.

.cortex/RECOVERY.md
  Safe resume process.

.cortex/WORKLOG.md
  Public decisions, evidence, and open questions.

.cortex/LEARNINGS.md
  Failed approaches, caveats, compatibility notes, and do-not-repeat lessons.

.cortex/SESSIONS.md
  Dated session history.

.cortex/TASKS.md
  Task backlog and completed work.

.cortex/GUIDE.md
  Operating guide.

.cortex/RULES.md
  Portable user and project rules.

.cortex/meta.json
  Machine-readable project index.

.cortex/mcp-tools.json
  Draft MCP tool contract.

prompts/
  Reusable prompt pack for installation, resume, closeout, and research work.

plugins/codex-cortex/
  Codex plugin package with the cross-project `codex-cortex-manager` skill.

plugins/codex-cortex/skills/codex-cortex-manager/
  System-installable Codex skill for installing and maintaining Cortex across
  any project. It also includes update-check and approval-gated update scripts.

plugins/codex-cortex/scripts/
  Plugin-level helper scripts, including system skill installation.

schemas/
  Machine-readable JSON Schemas for metadata and MCP tool contracts.

tools/Check-CodexCortexUpdates.ps1
  GitHub metadata and changelog update checker.

tools/Update-CodexCortexFromGitHub.ps1
  Approval-gated downloader and optional system skill installer.

publish/base/
  Generated project-free base for distribution and reuse.

Why These Names
---------------

The older prototype used longer names like CURRENT_STATE, RECOVERY_PLAYBOOK,
SESSION_LOG, and STREAM_OF_THOUGHT. Codex Cortex shortens those to STATE,
RECOVERY, SESSIONS, and WORKLOG.

The new names are easier to reference, less tied to one specific agent, and
more accurate for a reusable project-local continuity system.

Working State Versus Publish Base
---------------------------------

The root `.cortex/` folder belongs to this repository's own maintenance state.
It should not be treated as a blank template for external users.

The `template/` folder is the source blank template.

The `publish/base/` folder is generated from source files and should remain
project-free.
